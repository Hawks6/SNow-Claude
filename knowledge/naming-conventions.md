# Knowledge Base: Naming Conventions

> **File:** `naming-conventions.md`
> **Purpose:** Complete naming standard for every ServiceNow artifact type

---

## Application & Scope

| Item | Convention | Example |
|------|-----------|---------|
| Scope | `x_<vendor>_<app>` | `x_acme_itsm_ext` |
| Vendor prefix | 2-5 lowercase chars | `acme`, `myco` |
| App name | lowercase, underscores | `itsm_ext`, `hr_onboard` |

## Tables & Fields

| Item | Convention | Example |
|------|-----------|---------|
| Custom table | `x_<scope>_<name>` | `x_acme_app_approval_matrix` |
| Custom field | `u_<descriptive_name>` | `u_escalation_tier` |
| Reference field | `u_<referenced_table>` or descriptive | `u_approver`, `u_related_ci` |
| Choice field values | lowercase_underscore | `pending_approval` |

## Scripts

| Item | Convention | Example |
|------|-----------|---------|
| Script Include | `PascalCase` | `ApprovalMatrixUtils` |
| Ajax Script Include | `PascalCase` + `Ajax` suffix | `IncidentAjaxUtils` |
| Business Rule | `<Table> - <When> <Description>` | `Incident - Before Validate Resolution` |
| Client Script | `<Table> - <Type> <Description>` | `Incident - onChange Set Category Fields` |
| UI Policy | `<Table> - <Condition Description>` | `Incident - When VIP Show Priority` |
| UI Action | `<Table> - <Action Description>` | `Incident - Escalate to Manager` |
| Fix Script | `FIX - <TICKET#> - <Description>` | `FIX - STRY001 - Backfill Priority` |
| Scheduled Job | `<Scope> - <Frequency> <Description>` | `ITSM Ext - Daily Stale Incident Check` |

## Flows & Integration

| Item | Convention | Example |
|------|-----------|---------|
| Flow | `<Scope> - <Trigger> <Description>` | `ITSM Ext - Record Created Approval Flow` |
| Subflow | `<Scope> - SF <Description>` | `ITSM Ext - SF Route Approval` |
| Action | `<Scope> - <Verb> <Object>` | `ITSM Ext - Create JIRA Issue` |
| REST API | `<App> API` | `ITSM Extension API` |
| REST Resource | `/<version>/<resource>` | `/v1/incidents` |

## Security

| Item | Convention | Example |
|------|-----------|---------|
| Role | `x_<scope>.<role_name>` | `x_acme_app.approver` |
| ACL | Auto-generated by platform | `incident.priority (write)` |
| System Property | `x_<scope>.<category>.<name>` | `x_acme_app.approval.timeout_days` |

## Testing

| Item | Convention | Example |
|------|-----------|---------|
| ATF Suite | `ATF - <ClassName/Feature>` | `ATF - ApprovalMatrixUtils` |
| ATF Test | `ATF - <Target> - <Scenario>` | `ATF - ApprovalMatrixUtils - Happy Path` |

## Update Sets

| Item | Convention | Example |
|------|-----------|---------|
| Update Set | `[SCOPE] - [TICKET#] - [Description]` | `x_acme_app - STRY001 - Add approval matrix` |
| Batch Set | `[SCOPE] - [Release] - [Description]` | `x_acme_app - v1.2 - Sprint 5 Release` |

## Variables (Code)

```javascript
// JavaScript variables: camelCase
var approvalMatrix = {};
var isVipCaller = false;
var maxRetryCount = 3;

// Constants: UPPER_SNAKE (set in initialize)
this.MAX_RETRY_COUNT = 3;
this.LOG_PREFIX = '[ApprovalMatrixUtils]';
this.APPROVAL_TABLE = 'x_acme_app_approval_matrix';

// Private methods: prefixed with underscore
_validateInput: function(input) { ... }
_buildQuery: function(params) { ... }
```
